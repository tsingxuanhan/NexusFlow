## 核心模块

### CDoL 认知分工引擎（2,058 行）

六种视角分解策略 + 三轮有损通信协议 + FusionJudge 虚假一致检测。

核心创新：**主动制造信息不对称**，迫使每个 Agent 发展出从他人输出中逆向推断对方所见上下文的能力。

| 组件 | 职责 |
|------|------|
| PerspectiveDecomposer | 6种视角分解（全局/领域/方法/时间/利益相关者/反事实） |
| CommunicationLayer | 三轮有损通信（摘要传递 + 矛盾检测） |
| FusionJudge | 虚假一致检测 + 共识度收敛 |
| InsightDistiller | 跨任务经验蒸馏 |

### 自适应上下文管理器（1,642 行）

解决清华 & OpenBMB 发现的"大窗口懒惰症"——上下文越大，Agent 越倾向浅层推理。

### 三层信息架构（511 行）

AgentInformationPolicy：全局视野层 / CDoL 参与层 / 旁观记录层，为每个 Agent 分配角色化 ContextMask。

### 动态拓扑路由器（869 行）

运行时重建 Agent 协作图，支持五种拓扑模式：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| Sequential | 严格串行 | 流水线任务 |
| Parallel | 全并行 | 独立子任务 |
| Hybrid | 混合拓扑 | 复杂多阶段 |
| Star | 中心辐射 | 协调密集型 |
| Dynamic | 运行时自适应 | 不确定性任务 |

### 端边云三层调度器（535 行）

隐私优先调度，云端 DeepSeek API + 本地 Ollama 兜底。

### 统一编排器（479 行）

NexusOrchestrator：自动路由分类（simple / research / coding / cdol），任务完成后自动触发 Archivist 蒸馏归档。

## 设计理念

**框架工程 > 模型堆叠**

NexusFlow 不是又一个 Agent 框架，而是第一个实现**认知分工（Cognitive Division of Labor, CDoL）**的群体智能引擎。

不同于传统的任务分割式多 Agent 系统，NexusFlow 通过**主动制造信息不对称**，迫使每个 Agent 发展出从他人输出中逆向推断对方所见上下文的能力，从而产生超越任何单 Agent 的推理深度。

### 理论基础

- **Shannon 信道理论**：CDoL 2-3轮修正 = Nyquist采样下界（保证认知多样性的最低充分采样）
- **Simon 有限理性**：动态终止机制锁定 2-3 轮质量平台期
- **清华 & OpenBMB 大窗口懒惰症**：自适应上下文管理器的设计依据
- **Braintrust 实证**：框架对成功率的影响力是模型的 7.6 倍
