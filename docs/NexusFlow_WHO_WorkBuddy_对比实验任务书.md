> **版本**: v4.1 | **更新日期**: 2025-07-21

> **实验设计**: 训练-验证双表对照范式

---

## 1. 框架背景

### 1.1 NexusFlow 概述

NexusFlow 是一个多 Agent 协作框架，采用"端-边-云"三层架构与 CDoL（Cognitive Division of Labor）认知分工理念。系统由 10 个专职 Agent 构成，覆盖策略规划、知识获取、精确执行、质量审查、文献挖掘、知识验证、代码生成、领域专家、任务编排与记忆管理全链路。

核心设计原则：
- **认知分工**：不同 Agent 承担不同认知角色，模拟人类团队的专家协作
- **端边云路由**：简单任务在本地 Edge（Ollama）执行，复杂任务路由至 Cloud（DeepSeek API）
- **训练-验证范式**：通过历史数据训练 + 回测验证的双阶段实验，评估各 Agent 的分析与预测能力

### 1.2 十 Agent 分工映射

| 代号 | 名称 | 角色 | 模型层级 | 运行层 | 在本实验中的职责 |
|------|------|------|----------|--------|-----------------|
| Planner | 规划者 | 策略推理核心 | PRO | Cloud | 设计分析策略，分解实验任务，制定预测方案 |
| Researcher | 研究者 | 知识获取核心 | FLASH | Edge | 检索 WHO 指标元数据，验证数据完整性 |

### 1.3 实验范式

本实验采用**训练集-验证集**双表对照范式：
- **表A（训练集）**：2000-2016年历史数据，17年跨度，用于 Agent 学习模式、发现趋势
- **表B（验证集）**：2017-2021年实际数据，5年跨度，用于回测验证 Agent 的预测能力

Agent 仅可访问表A进行历史分析和趋势推断，然后对表B对应时段做出预测。预测结果与表B真值对比，评估 Agent 的分析-预测能力。

## 3. 实验任务定义

### 3.1 L1 层级：历史分析任务

L1 任务基于表A（2000-2016年）数据，要求 Agent 完成深度历史分析。

#### L1-1：趋势识别与模式发现

**任务描述**：分析55个国家在15个指标上的17年历史数据，识别以下模式：
1. **增长/衰退趋势**：哪些国家的健康指标持续改善？哪些在恶化？
2. **收敛/发散模式**：各国指标差距是缩小还是扩大？
3. **结构性转折点**：哪些年份出现了显著的趋势变化（如2008金融危机、2015 SDG目标生效）？
4. **跨国相关性**：哪些指标高度相关？（如预期寿命 vs 卫生支出）

**交付物**：
- 每国每指标的趋势方向标注（↑改善 / →平稳 / ↓恶化）
- 关键转折点年份列表
- 指标相关性矩阵

**负责 Agent**：Executor（数据处理）+ Researcher（信息检索）+ Reviewer（结果审查）

#### L1-2：国家分类与聚类

**任务描述**：基于17年健康数据对55个国家进行聚类分析：
1. 按健康水平分层（发达/发展中/欠发达）
2. 按改善速度分组（快速/中速/慢速改善）
3. 识别异常值（偏离同类国家平均表现的国家）

**交付物**：
- 国家分组结果
- 各组特征画像
- 异常值分析报告

**负责 Agent**：Executor（聚类计算）+ Planner（分组策略）+ Artisan（领域解读）

#### L1-3：驱动因素关联分析

**任务描述**：以卫生支出占GDP比例（GHED_CHEGDP_SHA2011）为自变量，分析其与14个健康结果指标之间的关联强度：
1. 哪些健康指标对卫生投入最敏感？
2. 投入-产出弹性系数估算
3. 边际效益递减效应识别

**交付物**：
- 投入-产出相关系数矩阵
- 回归分析报告
- 关键发现摘要

**负责 Agent**：Executor（统计分析）+ Miner（文献对标）+ Assayer（结论验证）

### 3.2 L2 层级：预测回测任务

L2 任务要求 Agent 基于表A知识，对表B（2017-2021）做出预测，然后与实际值对比。

#### L2-1：指标趋势外推

**任务描述**：对55个国家×15个指标×5年（2017-2021）共4,125个数据点，基于表A趋势进行预测：
1. 使用表A最后3年（2014-2016）的趋势斜率外推
2. 标注预测置信度（高/中/低）
3. 对有缺失数据的指标（疟疾、心血管）做特殊处理

**预测方法参考**：
- 线性趋势外推
- 移动平均法
- 基于同类国家均值推断

**交付物**：
- 4,125个数据点的预测值
- 预测置信度标注
- 预测方法论说明

**负责 Agent**：Caster（生成预测代码）+ Executor（执行预测）+ Planner（策略选择）

#### L2-2：异常事件预测

**任务描述**：识别表B中可能出现的异常波动：
1. 哪些国家-指标组合可能在2017-2021年间出现显著偏离趋势？
2. 哪些国家最可能实现/无法实现SDG健康相关目标？

**交付物**：
- 异常事件预测列表
- SDG目标达成概率评估

**负责 Agent**：Planner（策略推理）+ Researcher（背景检索）+ Artisan（领域判断）

#### L2-3：预测结果校验

**任务描述**：将预测结果与表B真值进行对比，计算评估指标：
1. 均方根误差(RMSE)
2. 平均绝对百分比误差(MAPE)
3. 趋势方向准确率（预测↑↓方向是否正确）
4. 国家排名保持率（排名变化 < 5位视为正确）

**交付物**：
- 逐指标预测精度报告
- 逐国家预测偏差分析
- 整体预测能力评分

**负责 Agent**：Reviewer（质量评估）+ Assayer（交叉验证）+ Archivist（经验归档）

## 4. 六维评估框架

对每个 Agent 的实验表现从以下六个维度进行评估：

### 4.1 评估维度定义

| 维度 | 代号 | 权重 | 说明 |
|------|------|------|------|
| 准确性 | Accuracy | 25% | 预测值与真值的接近程度（MAPE/RMSE） |
| 完整性 | Completeness | 20% | 是否覆盖了所有要求的分析维度 |
| 一致性 | Consistency | 15% | 多次分析/预测的结果是否稳定一致 |
| 时效性 | Timeliness | 10% | 任务完成时间是否在约束范围内 |
| 可解释性 | Interpretability | 15% | 分析结论是否有清晰的因果解释 |
| 协作效率 | Collaboration | 15% | 多 Agent 协作中的贡献度和冲突率 |

### 4.2 评分标准

| 等级 | 分数区间 | 含义 |
|------|----------|------|
| S | 90-100 | 卓越，超越基准线 |
| A | 75-89 | 优秀，达到预期 |
| B | 60-74 | 良好，基本达标 |
| C | 40-59 | 及格，有明显不足 |
| D | 0-39 | 不及格，需重大改进 |
