## 版本命名规则

- **主版本号** (3→4): 架构级变更，不可向后兼容
- **次版本号** (4.0→4.1): 新增核心模块，增量叠加
- **修订号** (3.1→3.1.1): 局部优化，无新模块

## 设计原则

1. **纯增量叠加**: 每个版本只新增文件，不修改上一版本已有代码
2. **桥接集成**: 新版本通过 `vX_integration.py` 统一初始化入口桥接
3. **代码快照**: 每个版本的 `code/` 是该版本的完整可运行状态
4. **设计先行**: 先写设计文档，再实现代码
