## Phase 1：紧急补强（1-2天，ROI最高）

> 目标：让评委 git clone 后 5 分钟内跑起来

### 1.1 Docker 一键部署 ⭐⭐⭐
**为什么紧急**：评委打开仓库看到 `docker compose up` 就能跑，和看到一堆 pip install 步骤，印象天差地别。

**交付物**：
- `Dockerfile`（多阶段构建，最终镜像 < 2GB）
- `docker-compose.yml`（nexusflow + dashboard + 可选 Ollama）
- `.dockerignore`
- README 中增加 Quick Start with Docker 段落

**具体方案**：
```dockerfile
# Dockerfile
FROM python:3.11-slim AS base
WORKDIR /app
COPY pyproject.toml .
RUN pip install --no-cache-dir .
COPY . .
EXPOSE 8000
CMD ["nexusflow", "serve"]
```

```yaml
# docker-compose.yml
services:
  nexusflow:
    build: .
    ports: ["8000:8000"]
    environment:
      - DEEPSEEK_API_KEY=${DEEPSEEK_API_KEY}
    volumes:
      - ./data:/app/data

  # 可选：本地 Ollama
  ollama:
    image: ollama/ollama:latest
    ports: ["11434:11434"]
    profiles: ["local-llm"]
```

### 1.2 Makefile ⭐⭐⭐
**为什么重要**：评委看到 Makefile 就知道这是个正经工程。

**交付物**：
```makefile
.PHONY: install test lint format serve demo clean docker

install:
	pip install -e ".[dev]"

test:
	python -m pytest tests/ -v --tb=short --timeout=30

test-cov:
	python -m pytest tests/ -v --cov=nexusflow --cov-report=term-missing

lint:
	ruff check nexusflow/ --fix
	ruff format nexusflow/

format:
	ruff format nexusflow/ tests/ examples/

serve:
	nexusflow serve

demo:
	nexusflow demo

doctor:
	nexusflow doctor

docker-build:
	docker build -t nexusflow:latest .

docker-up:
	docker compose up -d

clean:
	find . -type d -name __pycache__ -exec rm -rf {} +
	rm -rf .pytest_cache dist/ build/ *.egg-info
```

## Phase 2：代码拆分（2-3天）

> 目标：消灭所有 > 1000 行的单文件

### 2.1 base_agent.py 拆分（2597行 → 6个模块）

**当前结构分析**：base_agent.py 混合了 8 个独立关注点：

| 当前类 | 行数范围 | 职责 | 拆分目标 |
|--------|---------|------|---------|
| Message, AgentRole, AgentRunMode | 1-180 | 数据模型 | `agents/models.py` |
| TodoItem, TodoProvider | 181-260 | 任务管理 | `agents/todo_provider.py` |
| ContextCompactor | 262-365 | 上下文压缩 | `agents/context_compactor.py` |
| BaseAgent.__init__ ~ set_context_window | 367-610 | 核心初始化 | `agents/base_agent.py`（保留，瘦身后） |
| plan/execute_step/reflect | 610-870 | 推理链 | `agents/reasoning.py`（mixin） |
| init_codeact/execute_codeact | 918-1035 | 代码执行 | `agents/codeact_mixin.py`（mixin） |
| init_memory/remember/recall/dream | 1036-1190 | 记忆系统 | `agents/memory_mixin.py`（mixin） |
| a2a_protocol/checkpoint/rewind | 1210-end | 协作与持久化 | `agents/collaboration_mixin.py` |

**拆分策略**：使用 Mixin 模式，BaseAgent 组合多个 Mixin：
```python
# agents/base_agent.py（瘦身后 ~400行）
class BaseAgent(
    ReasoningMixin,
    CodeActMixin,
    MemoryMixin,
    CollaborationMixin,
):
    def __init__(self, ...):
        # 只保留初始化和基础通信
```

**兼容性**：通过 `__init__.py` 导出保持向后兼容：
```python
# agents/__init__.py
from .base_agent import BaseAgent
from .models import Message, AgentRole, AgentRunMode
from .todo_provider import TodoProvider
from .context_compactor import ContextCompactor
```

## Phase 3：CI/CD 完善（1天）

> 目标：GitHub 每个 PR 自动检查，评委看到绿 badge

### 3.1 增强现有 tests.yml
```yaml
name: Tests
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ["3.10", "3.11", "3.12"]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python-version }}
      - run: pip install -e ".[dev]"
      - run: pytest tests/ -v --tb=short --timeout=30 --cov=nexusflow --cov-report=xml
      - uses: codecov/codecov-action@v4
        with:
          file: ./coverage.xml
```

### 3.2 新增 lint.yml
```yaml
name: Lint
on: [push, pull_request]
jobs:
  ruff:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: chartboost/ruff-action@v1
        with:
          src: "./nexusflow"
```

### 3.3 新增 build.yml（发布流程）
```yaml
name: Build & Publish
on:
  push:
    tags: ["v*"]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: pip install build
      - run: python -m build
      - uses: actions/upload-artifact@v4
        with:
          name: dist
          path: dist/
  publish:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/download-artifact@v4
      - uses: pypa/gh-action-pypi-publish@release/v1
        with:
          password: ${{ secrets.PYPI_TOKEN }}
```

### 3.4 新增 Docker CI
```yaml
# 加到 docker-compose.yml 的 CI 验证
name: Docker
on: [push, pull_request]
jobs:
  docker:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: docker build -t nexusflow:test .
      - run: docker run --rm nexusflow:test nexusflow doctor
```

## Phase 4：代码质量工具链（半天）

> 目标：统一代码风格，自动检查

### 4.1 Ruff 配置（加入 pyproject.toml）
```toml
[tool.ruff]
target-version = "py310"
line-length = 120
src = ["nexusflow", "tests"]

[tool.ruff.lint]
select = ["E", "F", "W", "I", "N", "UP", "B", "SIM"]
ignore = ["E501"]  # 允许长行（学术代码常见）

[tool.ruff.lint.isort]
known-first-party = ["nexusflow"]
```

### 4.2 Pytest 配置
```toml
[tool.pytest.ini_options]
testpaths = ["tests"]
timeout = 30
markers = [
    "slow: marks tests as slow (deselect with '-m \"not slow\"')",
    "integration: marks integration tests",
]
```

### 4.3 pre-commit hooks（可选，评委会加分）
```yaml
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.4.0
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format
```

### 4.4 dev 依赖组（加入 pyproject.toml）

## Phase 5：API 文档生成（半天）

> 目标：评委能在线浏览 API 文档

### 5.1 方案选型
| 方案 | 优点 | 缺点 |
|------|------|------|
| MkDocs + mkdocstrings | GitHub Pages 自动部署，好看 | 需要额外配置 |
| Sphinx | 功能最全 | 配置复杂 |
| **pdoc** | 零配置，一行命令 | 定制性一般 |

**推荐 pdoc**（最快出活）：
```bash
pip install pdoc
pdoc nexusflow -o docs/api/ --logo "https://..."
```

然后 GitHub Pages 部署 `docs/api/`。

### 5.2 补充 docstring
优先为以下公共接口补充 Google-style docstring：
- `BaseAgent.__init__`, `plan`, `execute_step`, `reflect`
- `CognitiveDivisionEngine.run_full_pipeline`
- `DynamicTopologyRouter.route`
- `AdaptiveContextManager.manage`
- `EdgeCloudScheduler.schedule`

## 风险与约束

1. **base_agent.py 拆分风险**：改动面大，需要确保所有 examples/ 和 tests/ 的 import 不挂。拆分后必须全量回归。
2. **pytest 超时**：可能是某些测试直接调用真实 API，需要系统性地 mock。
3. **Docker 镜像大小**：依赖 litellm + gradio + aiohttp 等，可能较大。用 multi-stage build 控制。
4. **时间约束**：9月15日截止，Phase 1-4 建议在 1 周内完成（高 ROI），Phase 5-6 可选。
