## 1. 最终采用的评分方法

### 1.1 评估方法：10维度确定性规则评估

评分采用**确定性规则评估**（非LLM评估），基于预定义的评分规则对每个维度进行打分，确保评估的可重复性和客观性。

### 1.2 评分维度与权重

| 维度 | 权重 | 评分标准（满分10分） |
|------|:----:|------|
| 数据准确性 | 15% | 数值是否与WHO GHO API官方数据一致 |
| 排名正确性 | 15% | 综合健康指数排名是否正确 |
| 分析深度 | 15% | 是否有因果分析而非仅罗列数据 |
| 方法论 | 10% | 综合指数计算方法是否合理、是否说明计算逻辑 |
| 完整性 | 10% | 是否覆盖全部3项指标+排名+分析+建议 |
| 交叉验证 | 10% | 是否进行了多源数据比对 |
| 不确定性标注 | 5% | 是否标注数据时效性、指标代表性等局限 |
| 可操作性 | 5% | 政策建议是否具体、可落地 |
| 逻辑一致性 | 10% | 结论是否自洽、前后是否矛盾 |
| 可复现性 | 5% | 执行过程是否可复现 |

**加权总分计算**：`总分 = Σ(维度得分 × 权重) × 10`（满分100分）

### 1.3 评估基准

所有框架使用完全相同的任务和数据基准：
- **统一LLM**: DeepSeek Chat (deepseek-chat)
- **统一数据源**: WHO GHO OData API
- **统一任务**: BRICS五国健康指标分析（预期寿命、婴儿死亡率、卫生支出）
- **基准数据**: 预置的WHO官方数据用于验证输出准确性

以下文件为评估过程中的中间产物，记录了不同阶段、不同方法的评分结果。**它们不代表最终结论，仅供追溯评估过程时参考**。

### 分数差异说明

不同版本分数差异主要来源于：
1. **评估任务不同**：早期版本基于简化任务，后续使用完整WHO数据分析任务
2. **评估方法迭代**：从LLM主观评分演进为确定性规则评估，提高了可重复性
3. **评分维度变化**：从5维（completeness/depth/consistency/novelty/actionability）扩展为10维
4. **已知异常**：`comparison_results.json` 中 AutoGen=100 为评估脚本bug（所有维度被错误赋满分10），已排除

在论文、报告或其他文档中引用本实验分数时：

1. **必须引用权威版本**：`multi_framework_comparison.json` 中的分数
2. **必须注明评估方法**：「10维度确定性规则评估」
3. **必须注明框架版本**：NexusFlow v2.9.1、AutoGen v0.7.5
4. **必须注明运行模式**：真实执行 vs 模拟执行
5. **禁止引用**：`comparison_results.json` 中 AutoGen=100 的结果
