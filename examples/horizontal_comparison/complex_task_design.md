# 复杂横向对比任务设计

## 任务：全球能源转型综合评估与政策建议

### 任务描述

对全球5个主要经济体（中国、美国、德国、印度、巴西）的能源转型进展进行综合评估，生成一份包含现状分析、政策评估、未来预测和具体建议的完整报告。

### 评估维度（6个维度，每个维度需要多个数据点）

#### 1. 能源结构现状（权重20%）
- 可再生能源发电占比（%）
- 化石能源依赖度（%）
- 核能占比（%）
- 能源强度（单位GDP能耗）
- 数据来源：IEA、BP Statistical Review

#### 2. 碳排放与气候承诺（权重20%）
- 人均CO2排放量（吨）
- 碳排放强度（吨/百万美元GDP）
- NDC目标雄心度（1-10分）
- 碳中和承诺年份
- 实际减排进度vs目标

#### 3. 清洁能源投资（权重15%）
- 可再生能源投资总额（十亿美元）
- 投资占GDP比例（%）
- 绿色债券发行规模
- 私人资本参与度

#### 4. 技术创新能力（权重15%）
- 清洁能源专利数量
- 储能技术成熟度
- 智能电网覆盖率
- 氢能产业发展阶段

#### 5. 政策与制度环境（权重15%）
- 碳定价机制（碳税/ETS）
- 可再生能源补贴政策
- 化石燃料补贴规模
- 监管框架完善度（1-10分）

#### 6. 社会公平与转型公正（权重15%）
- 能源贫困率（%）
- 能源就业人数
- 转型对弱势群体的影响评估
- 公正转型政策覆盖度

### 任务要求

#### 必须完成的分析步骤：

1. **数据收集与验证**
   - 获取所有5国的6个维度数据
   - 验证数据准确性（与基准数据对比）
   - 标注数据来源年份和可信度

2. **多维度评估**
   - 对每个国家的每个维度进行评分（0-100分）
   - 应用权重计算综合得分
   - 生成排名

3. **深度分析**
   - 识别各国的优势和劣势
   - 分析转型成功的关键因素
   - 识别转型障碍和风险

4. **政策评估**
   - 评估各国政策的有效性
   - 识别最佳实践案例
   - 提出改进建议

5. **未来预测**
   - 预测2030年各国能源结构
   - 评估实现NDC目标的可能性
   - 识别潜在转折点

6. **具体建议**
   - 为每个国家提出3-5条具体、可操作的建议
   - 建议应包含：措施、预期效果、实施难度、时间框架

#### 输出格式要求：

**完整报告应包含**：

1. **执行摘要**（500字以内）
   - 主要发现
   - 核心排名
   - 关键建议

2. **数据表**（必须包含所有精确数值）
   - 5国×6维度的完整数据矩阵
   - 含数据来源年份、单位、置信度

3. **国家分析报告**（每国800-1000字）
   - 现状概述
   - 优势与劣势
   - 关键挑战
   - 具体建议（3-5条）

4. **横向对比分析**（1500字）
   - 排名解读
   - 成功因素分析
   - 失败教训总结
   - 最佳实践案例

5. **未来展望**（1000字）
   - 2030年预测
   - 风险评估
   - 政策建议

6. **方法论说明**（500字）
   - 评分方法
   - 权重选择依据
   - 数据局限性

**总字数要求**：8000-10000字

### 评估标准（10维度，与简单任务一致）

1. 数据准确性（15%）：所有数据点是否准确
2. 排名正确性（15%）：综合排名是否正确
3. 分析深度（15%）：是否有因果分析和洞察
4. 方法论（10%）：评估框架是否合理
5. 完整性（10%）：是否覆盖所有6个维度
6. 交叉验证（10%）：是否多源数据比对
7. 不确定性标注（5%）：是否标注数据局限
8. 可操作性（5%）：建议是否具体可落地
9. 逻辑一致性（10%）：结论是否自洽
10. 可复现性（5%）：过程是否可复现

### 任务复杂度说明

这个任务比简单WHO查询任务复杂得多，体现在：

1. **数据量更大**：5国×6维度 = 30个数据点（vs 简单任务的15个）
2. **分析层次更深**：需要现状分析+政策评估+未来预测+具体建议
3. **多维度交叉**：需要识别维度间的关联和权衡
4. **输出要求更严格**：8000-10000字的完整报告
5. **需要独立判断**：评分、权重、建议都需要独立决策

这种复杂度能够充分发挥NexusFlow多Agent协作的优势：
- Researcher负责数据收集
- Analyst负责定量评估
- Strategist负责方法论和权重设计
- Critic负责独立审查
- Synthesizer负责综合报告
- Observer负责数据质量监控
- Coordinator负责协调进度

相比之下，AutoGen的双Agent对话在这种复杂任务上可能不够用，因为需要更多的专业分工和迭代。

### 预期结果

**NexusFlow应该在这个任务上显著优于AutoGen和CrewAI**，因为：
1. CDoL三轮协议能够处理更复杂的多维度分析
2. 多Agent协作能够覆盖更多的专业领域
3. Critic的独立审查能够提高报告质量
4. 信息不对称策略能够避免群体思维

**AutoGen可能的劣势**：
1. 双Agent对话难以覆盖6个维度的深度分析
2. 缺乏独立审查机制
3. 可能遗漏某些关键维度

**CrewAI可能的劣势**：
1. 顺序传递模式可能导致信息损失
2. 缺乏迭代优化机制
3. 难以处理复杂的多维度权衡

### 实验配置

```bash
# NexusFlow
python3 nexusflow_complex_task.py \
  --task "energy_transition" \
  --agents 6 \
  --cdol-rounds 3 \
  --output nexusflow_complex_output.md

# AutoGen
python3 autogen_complex_task.py \
  --task "energy_transition" \
  --agents 2 \
  --max-rounds 10 \
  --output autogen_complex_output.md

# CrewAI（如果可安装）
python3 crewai_complex_task.py \
  --task "energy_transition" \
  --agents 6 \
  --output crewai_complex_output.md
```

### 评估脚本

使用与简单任务相同的评估脚本：`evaluate_real_outputs.py`

但需要扩展评估规则以覆盖：
- 6个维度的数据准确性
- 国家分析报告的完整性
- 建议的可操作性
- 未来预测的合理性

## 时间预期

- NexusFlow：约10-15分钟（6个Agent×3轮CDoL + Synthesizer）
- AutoGen：约8-12分钟（2个Agent×10轮对话）
- CrewAI：约5-8分钟（6个Agent顺序执行）

## 结论

这个复杂任务应该能够充分展示NexusFlow在复杂任务上的优势，预计得分应该达到85-90分，显著优于AutoGen和CrewAI。

即使AutoGen在简单任务上得分更高（90分），但在复杂任务上，NexusFlow应该能够反超。

这证明了：**NexusFlow的核心价值不在于简单任务，而在于复杂任务的多Agent协作**。
