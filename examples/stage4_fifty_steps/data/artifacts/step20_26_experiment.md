### **实验方案深化设计 (步骤20-26)**

#### **20. 数据分布分析 (Data Distribution Analysis)**

**目标：** 理解关键变量的统计特征，为后续模型选择（如DLNM中的基函数）和敏感性分析提供依据。

**操作方案：**

1.  **NOAA气象数据 (辛辛那提)：**
    *   **变量：** `TMAX`, `TMIN`, `PRCP`, `AWND` (平均风速), `WDIR` (风向，需转换为连续变量如正弦/余弦分量)。
    *   **分析：**
        *   **直方图与Q-Q图：** 检查 `TMAX`, `TMIN` 是否近似正态分布。极端高温/低温通常呈偏态分布。
        *   **时间序列图：** 绘制 `TMAX` 和 `PRCP` 的月度/年度趋势，识别季节性模式和长期趋势（如年均温是否上升）。
        *   **缺失值模式：** 使用 `VIM` 包（在R中）或 `missingno`（在Python中）可视化缺失值分布。判断缺失是随机（MAR）还是非随机（MNAR）。例如，如果某个月份的 `PRCP` 数据全部缺失，可能为传感器故障（非随机）。
    *   **处理：** 若为MAR，采用多重插补（MICE）；若为MNAR，需在敏感性分析中讨论其影响。

2.  **WHO健康数据 (全球生命期望)：**
    *   **变量：** `Life expectancy`, `Adult Mortality`, `infant deaths`, `Alcohol`, `GDP`, `Schooling`。
    *   **分析：**
        *   **箱线图：** 按年份或WHO区域（如非洲、欧洲）绘制 `Life expectancy` 的箱线图，识别异常值（如某些国家因战争或疫情导致的异常下降）。
        *   **相关性矩阵：** 计算 `Life expectancy` 与 `GDP`, `Schooling`, `Alcohol` 的皮尔逊相关系数。预期 `GDP` 和 `Schooling` 与生命期望正相关。
        *   **面板数据结构：** 确认数据是平衡面板（每个国家每年都有数据）还是非平衡面板。样本显示为平衡面板。
    *   **处理：** 对于 `infant deaths` 等计数变量，检查是否存在过度离散（方差远大于均值），若存在，则需在后续模型中考虑负二项分布。

#### **21. DLNM模型设计 (Distributed Lag Non-linear Model Design)**

**目标：** 精确建模气候变量（如温度、降水）对健康结局（如特定疾病死亡率）的非线性和滞后效应。

**操作方案：**

1.  **核心模型结构 (以温度-死亡率为例)：**
    *   `Y_t ~ Poisson(μ_t)` 或 `Negative Binomial(μ_t, θ)` (当数据过度离散时)
    *   `log(μ_t) = α + cb(Temp_t, lag) + ns(Time_t, df*7/year) + ns(Humidity_t, 3) + DOW_t + Holiday_t + offset(Pop_t)`
    *   **解释：**
        *   `Y_t`：第 `t` 天的死亡人数（或发病率）。
        *   `cb(Temp_t, lag)`：交叉基函数，同时建模温度的非线性效应和滞后效应。使用 `dlnm` 包（R语言）。
            *   **温度维度：** 采用 `ns` (自然立方样条) 或 `bs` (B样条)，自由度 `df_temp = 3-5`，节点放置在温度分布的百分位数上（如10th, 50th, 90th）。
            *   **滞后维度：** 采用 `ns` 或 `bs`，自由度 `df_lag = 3-5`，最大滞后天数 `L = 21` 或 `30`（基于H2假设）。
        *   `ns(Time_t, df*7/year)`：控制长期趋势和季节性。自由度 `df` 通常选择 `3-7` /年。例如，`7` /年允许模型捕捉到季节性波动。
        *   `ns(Humidity_t, 3)`：控制湿度的影响。
        *   `DOW_t`：星期几的哑变量。
        *   `Holiday_t`：公共假日的哑变量。
        *   `offset(Pop_t)`：人口规模的偏移量。

2.  **空间模型 (针对全球数据)：**
    *   若使用WHO的全球面板数据，可采用**多水平DLNM**或**两阶段分析**：
        *   **第一阶段：** 对每个国家/地区单独拟合上述DLNM，得到国家特定的温度-死亡率关联。
        *   **第二阶段：** 使用多元Meta回归，将国家特定效应（如最小死亡温度MMT）与国家级协变量（如GDP、医疗支出、纬度）进行关联。

#### **23. 敏感性方案 (Sensitivity Analysis Plan)**

**目标：** 检验核心结果是否依赖于模型假设或数据选择。

**方案A：改变模型自由度。**
*   **操作：** 将温度样条的自由度从 `df_temp=4` 改为 `3` 和 `5`；将时间趋势的自由度从 `7/年` 改为 `5/年` 和 `9/年`。
*   **评估：** 观察温度-死亡率关联曲线的形状和置信区间是否发生根本性变化。如果结果稳健，则增强信心。

**方案B：改变滞后天数。**
*   **操作：** 将最大滞后天数 `L` 从 `21` 天改为 `14` 天和 `28` 天。
*   **评估：** 检查滞后效应的累积风险是否在 `L=21` 时已完全捕获。

**方案C：排除极端事件。**
*   **操作：** 剔除数据中极端高温/低温年份（如2012年热浪）或疫情年份（如COVID-19大流行期）。
*   **评估：** 核心关联是否由少数极端事件驱动。

**方案D：使用不同数据源。**
*   **操作：** 将NOAA的气温数据替换为ERA5（欧洲中期天气预报中心）再分析数据。
*   **评估：** 暴露测量误差对结果的影响。

#### **24. Critic审查 (Critical Review by Critic Agent)**

**角色：** Critic Agent（由我模拟）将基于科学严谨性、逻辑一致性和可行性提出质疑。

**Critic Agent审查意见：**

1.  **数据粒度不匹配问题：**
    *   **质疑：** 您设计的DLNM模型需要每日/每周的健康结局数据，但WHO样本提供的是**年度**国家层面的数据。这是一个严重的粒度不匹配。年度数据无法捕捉滞后几天的急性效应。
    *   **建议：** **必须**获取更细粒度的健康数据。对于美国本土，可尝试申请**CDC WONDER**（按县、按周的死亡率数据）或**HCUP**（住院数据）。对于全球分析，可考虑**GBD (Global Burden of Disease)** 数据，但其时间分辨率通常为年。

2.  **混杂控制不足：**
    *   **质疑：** 方案中控制了湿度，但未提及**空气污染**（PM2.5, O3）和**流感活动**。空气污染是重要的混杂因素，且与气象条件相关。
    *   **建议：** 必须将PM2.5和流感活动数据纳入模型。若无法获取，需在讨论部分明确指出此局限性，并进行敏感性分析（如假设PM2.5的潜在影响范围）。

3.  **空间异质性问题：**
    *   **质疑：** H3假设提到空间异质性，但当前方案主要聚焦于辛辛那提一个城市。一个城市无法代表全球或国家层面的异质性。
    *   **建议：** 扩展分析至多个气候带（如美国：凤凰城、西雅图、迈阿密），或使用多城市/多国家Meta分析框架。
