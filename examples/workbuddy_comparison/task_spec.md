## §1 背景：NexusFlow框架是什么

### 1.1 一句话定位

**NexusFlow** 是一个面向超长程复杂任务的**动态异构群体智能架构**（Dynamic Heterogeneous Swarm Intelligence Architecture）。它不是一个更大的LLM，而是一套让多个异构Agent高效协作的**协议框架**。

### 1.2 核心理念

**框架工程 > 模型堆叠**。NexusFlow 的核心信念是：复杂任务的瓶颈不在于单个模型有多强，而在于如何组织多个专业化Agent的信息流、决策流和纠错流。类比来说——NexusFlow 不是造一个更强的士兵，而是设计一套让10个不同专长的士兵高效协同作战的指挥系统。

### 1.3 核心机制：CDoL（Cognitive Data Orchestration Layer）

**CDoL（认知数据编排层）** 是 NexusFlow 独创的信息流控制机制，也是区别于其他多Agent框架的关键差异点。

**CDoL 解决的核心问题**：在多Agent协作中，每个Agent应该看到什么数据？看到太多会信息过载、角色混淆；看到太少会遗漏关键信息、决策偏颇。

**CDoL 的运作方式**：
- **角色感知数据切片**：每个Agent只能访问与自己角色定义相关的数据子集。例如 Researcher 负责获取原始数据，Executor 负责基于 Researcher 提供的清洗后数据执行分析，Reviewer 只看最终输出和中间结论。
- **全局共识度（Global Consensus Score）**：一个0-1的浮点数，表示当前所有Agent对某个决策点的共识程度。每轮迭代后更新，低于阈值（默认0.7）的议题必须继续讨论。
- **上下文传播协议**：Agent之间的信息传递不是简单的消息广播，而是经过CDoL编排的结构化上下文包（Context Packet），包含：任务目标摘要、前序Agent的关键结论、相关数据子集、当前共识度。

### 1.4 10个Agent的角色定义

NexusFlow 的10个Agent分为两种层级：

- **PRO（专业级）**：处理需要深度推理的复杂任务，运行在CLOUD层
- **FLASH（快速级）**：处理高频、标准化的执行任务，运行在EDGE层（延迟更低）

| Agent | 角色 | 核心职责 | Model Tier | Run Layer |
|---|---|---|---|---|
| **Coordinator** | 多Agent协作调度中枢 | 负责任务分解、Agent调度、冲突仲裁、进度监控。是整个系统的"大脑中的大脑"。 | PRO | CLOUD |
| **Planner** | 策略推理核心 | 负责将模糊的高层目标转化为可执行的步骤序列。处理"做什么"和"为什么做"。 | PRO | CLOUD |
| **Researcher** | 知识获取与验证核心 | 负责从数据源获取原始信息，进行初步清洗和结构化。处理"数据从哪来"。 | FLASH | EDGE |
| **Executor** | 精确执行核心 | 按照Planner的步骤序列，执行具体的计算、分析、生成操作。处理"怎么做"。 | FLASH | EDGE |
| **Reviewer** | 质量守门员 | 对所有Agent的输出进行质量审查，标记不一致、错误或遗漏。处理"做得对不对"。 | FLASH | EDGE |
| **Miner** | 文献挖掘专家 | 从海量数据/文档中提取关键模式、趋势、异常。擅长发现隐藏相关性。 | PRO | CLOUD |
| **Assayer** | 知识验证专家 | 对Miner/Researcher的发现进行独立交叉验证。用不同方法验证同一结论。 | FLASH | CLOUD |
| **Caster** | 代码生成专家 | 将分析需求转化为可执行代码（Python/R/SQL），并执行计算密集型任务。 | PRO | CLOUD |
| **Artisan** | 领域专家 | 提供宏观经济学的领域知识，判断分析结果的经济学合理性。 | FLASH | EDGE |
| **Archivist** | 知识守护者与记忆管理者 | 管理长期记忆、维护知识库、确保信息不丢失不重复。 | FLASH | EDGE |

### 1.5 信息流协议（简化版）

```
[用户任务] 
    → Coordinator（分解任务）
        → Planner（制定分析策略）
            → Researcher + Miner（并行获取数据与文献）
                → Assayer（交叉验证发现）
                    → Executor + Caster（执行计算与分析）
                        → Artisan（领域知识审核）
                            → Reviewer（质量审查）
                                → Archivist（归档结论）
    → Coordinator（汇总共识度，决定是否需要迭代）
        → 共识度 ≥ 0.7 → 输出最终结果
        → 共识度 < 0.7 → 回到Planner重新规划
```

## §2 实验目标

### 2.1 核心实验问题

> **在同一个复杂的宏观经济分析+预测任务上，10Agent NexusFlow 协作模式相比单Agent独立模式，在分析质量、预测精度、洞察深度等维度上是否存在显著优势？优势的具体来源是什么？**

### 2.2 实验组设计

| 组别 | 模式 | 描述 |
|---|---|---|
| **A组（基线）** | 单Agent | 一个LLM Agent独立完成全部L1+L2分析任务。可以类比为一个"全能分析师"——没有角色切换、没有协作协议、没有共识度机制。 |
| **B组（实验）** | NexusFlow 10Agent | 10个Agent严格按§1.4的角色定义和§1.5的信息流协议协作完成任务。可以由一个LLM扮演多个角色（因为实验环境限制），但**必须严格遵循角色协议和信息流约束**——即每个"角色"只能访问其CDoL授权的数据子集，必须通过结构化Context Packet传递信息。 |

### 2.3 关键控制变量

- **同一个LLM底座**：两组使用相同的底层模型（如GPT-4o、Claude 3.5 Sonnet等），确保差异来自框架而非模型能力。
- **同一份数据**：两组看到完全相同的数据（表A+表B），但NexusFlow组的数据访问经过CDoL编排。
- **同一个任务Prompt**：L1任务的指令文本完全一致（见§4），区别只在于"一个Agent做"还是"10个Agent分工做"。

## §4 双层任务设计

### 4.1 L1 历史分析任务（基于表A，1980-2020）

#### 任务Prompt（两组完全一致）

```
你是一位受IMF委托的高级宏观经济分析师。请基于1980-2020年20个主要经济体的15项宏观经济指标数据，完成以下分析：

1. 识别每个国家在这一时期的3个关键结构性转折点（structural breaks）及其主要驱动因素。
   - "结构性转折点"指经济趋势发生根本性变化的年份（如金融危机、政策转型、外部冲击），而非普通波动。
   - 每个转折点需说明：哪一年、哪个/哪些指标发生转折、转折方向、可能的驱动因素。

2. 对比20国在以下三个维度的表现，给出综合评分排名（1-20名）：
   - 财政健康（Fiscal Health）：综合政府净债务/GDP、政府净借贷/GDP、政府总债务/GDP
   - 外部平衡（External Balance）：综合经常账户/GDP、进口增速、出口增速
   - 价格稳定（Price Stability）：综合通胀率、产出缺口
   - 每个维度给出0-100分的评分，综合排名取三维度加权平均。

3. 找出至少5组跨国相关性规律（cross-country correlation patterns）：
   - 例如：A国的X指标领先B国的Y指标N年
   - 例如：多国在某指标上呈现同步周期
   - 每组规律需说明：涉及哪些国家、哪些指标、相关系数（或定性描述强弱）、时间窗口、可能的传导机制。

4. 基于历史模式，对每个国家2021-2025年的以下5项关键指标给出区间预测：
   - 实际GDP增速（NGDP_RPCH）
   - 人均GDP（NGDPDPC）
   - 平均消费者价格通胀（PCPIPCH）
   - 失业率（LUR）
   - 政府净债务/GDP（GGXWDN_NGDP）
   
   每个预测需包含：
   - 预测区间（如"2.5%-3.5%"）
   - 置信度（高/中/低）
   - 主要驱动因素（2-3句话）
   - 主要风险因素（1-2句话）
```

### 4.2 L2 预测回测任务（对照表B真值）

在Agent完成L1任务后，将L1中的预测结果（任务4）与表B的真实数据进行对比回测：

#### 回测指标

| 指标 | 计算方式 | 说明 |
|---|---|---|
| **区间命中率（Hit Rate）** | 预测区间包含真实值的比例 | 20国×5指标×5年=500个数据点中，有多少被预测区间覆盖 |
| **MAPE** | mean(|预测中值-真实值|/真实值)×100% | 平均绝对百分比误差，越低越好 |
| **区间宽度（Interval Width）** | 预测区间上界-下界的平均值 | 越窄越好（但需保证命中率≥60%） |
| **驱动因素准确度** | 人工评估 | 对驱动因素分析的合理性打分（1-5分），抽样评估10国 |

#### 回测计算方法

对于每个国家、每个指标、每一年：
1. 检查真实值是否落在Agent给出的预测区间内 → 命中=1，未命中=0
2. 计算 MAPE = |预测中值 - 真实值| / |真实值| × 100%（真实值为0时跳过）
3. 汇总所有500个数据点的命中率和平均MAPE

## §5 评估维度（必须全部评估）

### 5.1 六维评估框架

| 维度 | 编号 | 权重 | 评估对象 | 评分标准（1-10分） |
|---|---|---|---|---|
| **信息完整性** | D1 | 15% | L1输出 | 10分=覆盖全部20国×15指标，无遗漏；每遗漏1国扣0.5分，每遗漏1组指标扣0.25分 |
| **分析深度** | D2 | 20% | L1任务1+2 | 10分=准确识别结构性转折点并给出经济学解释；5分=只描述趋势未识别转折点；1分=仅罗列数据 |
| **洞察质量** | D3 | 20% | L1任务3 | 10分=≥8组高质量跨国相关性，有传导机制解释；7分=5-7组合理规律；3分=仅1-2组弱相关 |
| **预测精度** | D4 | 20% | L2回测 | 10分=命中率≥80%且MAPE<5%；7分=命中率≥60%或MAPE<10%；3分=命中率<40%或MAPE>20% |
| **协作效率** | D5 | 15% | 仅B组 | 10分=共识度3轮内收敛至≥0.8，修正率>50%；5分=5轮收敛；1分=未收敛或无修正 |
| **可复现性** | D6 | 10% | A+B组 | 10分=3次运行结果一致率>90%；7分=一致率>70%；3分=一致率<50% |

### 5.2 协作效率详细指标（仅NexusFlow组）

以下指标仅在B组（NexusFlow模式）中评估：

#### 5.2.1 共识度收敛曲线

- **定义**：每轮迭代后，所有参与Agent对当前结论的共识度评分（0-1）
- **采集方式**：每轮迭代结束时，Coordinator汇总各Agent的结论一致性评分
- **期望形态**：初始共识度0.3-0.5 → 2-3轮后收敛至0.7-0.9
- **绘图要求**：X轴=迭代轮次，Y轴=全局共识度，曲线应展示收敛趋势

#### 5.2.2 信息流效率

- **定义**：CDoL编排下的有效数据访问次数 / 单Agent模式下的总数据访问次数
- **计算**：统计NexusFlow模式下各Agent实际访问的数据行数，与单Agent一次性加载全部数据的对比
- **期望**：NexusFlow的总数据访问量 < 单Agent（因为CDoL做了数据切片，避免冗余访问）

#### 5.2.3 错误自我修正率

- **定义**：Reviewer/Assayer在协作过程中发现并成功纠正的错误数 / 总错误数
- **采集方式**：记录每轮Reviewer的审查意见和修正建议
- **期望**：修正率 > 30%（即10个错误中至少修正3个）
