# xuanshu-agents 向量记忆系统升级 v3.1.1

## 升级时间
- 2026-06-01 完成

## 升级背景
`vector_memory.py` 的 `SimpleEmbeddingProvider` 用 hash 做向量，本质上等于随机搜索，完全没有语义理解。"nano SiO2 cement" 和 "nano silica concrete" 会得到完全无关的向量。

## 环境约束
- 云电脑 3.8GB RAM，当前可用 ~700MB
- HuggingFace 外网不通，无法下载模型
- chromadb 依赖太多装不上（已试过超时）
- DeepSeek API 可达（通过本地代理 127.0.0.1:8083）
- 已安装：numpy 2.4.3, scipy 1.17.1, onnxruntime 1.24.4

## 新增组件

### 1. NGramTFIDFProvider
**核心升级，替换 SimpleEmbeddingProvider 默认地位**

- 使用 word-level 和 character-level 的 n-gram 组合：
  - Word unigrams/bigrams: ["nano", "sio2", "cement", "nano_sio2", "sio2_cement"]
  - Char 3-grams: ["nan", "ano", "sio", "io2", "cem", "eme", "men", "ent"]
  - Char 4-grams: ["nano", "sio2", "ceme", "emen", "ment"]
- TF-IDF 加权：维护 IDF 字典，随语料增长自动更新
- 维度：固定 vocab size（默认 5000）
- 支持中英文混合（材料科学文献常见）
- **关键优势**：n-gram 能捕获子词相似性（"SiO2" 和 "silica" 共享 char n-gram）

### 2. APIKeywordProvider（可选）
用 DeepSeek flash 提取语义关键词，再转向量。用于高价值记忆（importance >= 0.7）

### 3. PersistentVectorStore
持久化向量存储：
- 保存到 `./xuanshu-agents/data/vector_memory.json`
- 每 10 次 add 自动保存
- 使用 tempfile 原子写入，避免数据损坏
- 启动时自动加载

### 4. ChromaDBVectorStore（可选后端）
- try/except 导入 chromadb
- 不可用时 graceful fallback 到 PersistentVectorStore
