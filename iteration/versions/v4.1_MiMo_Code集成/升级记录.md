# xuanshu-agents v4.1 升级 2026-06-12
## 背景
小米2026-06-11发布开源MiMo Code V0.1项目，基于OpenCode二次开发，MIT协议，内置MiMo-V2.5多模态模型，支持100万Token上下文窗口。经过深度调研对比，确定不直接替换当前使用的Codex链路，仅借鉴其三个优秀设计点融入现有xuanshu-agents框架。
## 不直接替换Codex链路的研判结果
1. 产品定位不匹配：MiMo Code是面向人类用户的终端交互式编程TUI工具，不是可直接导入Python作为依赖的通用库，无法适配当前自动化Agent框架的调用逻辑
2. Token成本过高：MiMo Code的Max模式5候选并行选优机制，每步Token消耗是现有Codex链路的4-5倍，会大幅超出积分预算
3. 硬件资源不足：当前云电脑仅3.8GB RAM，无法运行MiMo Code依赖的Bun/Node运行时环境
4. 版本稳定性差：V0.1刚发布，仅7个commit、3位贡献者，共165个未解决issue，生产环境可用性不足
## 具体改造内容
1. 新增`checkpoint_writer.py`：实现独立子Agent后台异步写入11字段结构化状态，在上下文利用率达20%/45%/70%三个节点触发增量更新，上下文接近上限时可自动从持久化记录重建新运行窗口
2. 新增`goal_verifier.py`：独立目标验证judge模块，主Agent试图终止任务时先由该模块验证是否真的满足完成条件，避免"乐观提前结束"的问题
3. 修改`continuous_learning.py`：新增7天周期的dream_cycle机制、文件路径有效性校验逻辑、学习记录去重压缩功能，优化原有Sleeptime整合流程
4. 修改`base_agent.py`：将两个新增模块集成到主Agent运行流中
