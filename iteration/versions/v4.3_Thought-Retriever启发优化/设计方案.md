## 一、设计动机

### 1.1 v4.2 已解决的问题
- ✅ 领域知识系统化注入（Knowledge Library）
- ✅ 假说驱动科学探索（Hypothesis Engine）
- ✅ 衍生发现链式挖掘（Discovery Exploitation）
- ✅ 内部修复循环（Self-Healing Loop）
- ✅ 通信极简化（Succinct Communication）
- ✅ 结构化模式记忆（Structured Pattern Memory）

### 1.2 v4.2 尚未解决的问题（Thought-Retriever 视角）

| 问题 | 现状 | 影响 |
|------|------|------|
| **存结论不存推理** | 记忆存储的是"事实+结论"，丢失了"怎么得出结论"的推理链 | 相似问题无法复用推理路径，每次都从头推理 |
| **无抽象层级匹配** | 所有记忆条目同等对待，不区分"浅层事实"和"深层策略" | 简单问题检索到过度复杂的策略，复杂问题只用浅层事实 |
| **沉淀滞后** | Dream 7天周期 + Distill 30天周期，沉淀延迟大 | 当天的高价值推理成果次日才可能被整理，可能已被遗忘 |
| **无置信度门控** | 新记忆写入时不做逻辑一致性校验 | 可能存入低质量或矛盾的"思想"，污染记忆库 |

### 1.3 Thought-Retriever 的核心洞察

> **"记忆不是数据的堆砌，而是思想的沉淀；智能不是静态的能力，而是动态的进化。"**

Thought-Retriever 证明了：
- 检索"验证过的推理链"比检索"原始数据"效果好得多
- 给记忆标注抽象层级，可以精准匹配任务复杂度
- 每次交互后即时提炼新思想，比周期性沉淀更高效
- 双重过滤（置信度+冗余）从源头抑制幻觉

## 三、高优先级模块

### 3.1 Thought Crystallization（思想结晶）

#### 3.1.1 设计目标
- 在每次 Agent 完成推理后，即时提炼一条"思想结晶"
- 思想结晶 ≠ 结论，而是**"结论 + 推理路径 + 关键决策点 + 适用条件"**的浓缩
- 目标：让未来的相似问题能复用整条推理路径，而非只复用结论

#### 3.1.2 数据结构

```python
@dataclass
class ThoughtDiamond:
    """思想结晶 — 验证过的推理链浓缩单元"""
    
    # 基础标识
    thought_id: str
    timestamp: float
    
    # 核心内容
    conclusion: str              # 最终结论（一句话）
    reasoning_chain: List[str]   # 推理步骤序列（每步一句话）
    key_decisions: List[str]     # 关键决策点（为什么选A不选B）
    
    # 适用条件（绑定上下文）
    problem_type: str            # 问题类型标签
    constraints: List[str]       # 约束条件
    assumptions: List[str]       # 假设前提
    
    # 元数据
    confidence: float            # 置信度 0.0-1.0
    abstraction_level: int       # 抽象层级 1-4（由 AbstractionHierarchy 标注）
    source_task_id: str          # 来源任务ID
    source_agent: str            # 来源Agent角色
    verification_status: str     # "verified" / "pending" / "rejected"
    
    # 关联
    related_thoughts: List[str]  # 相关思想ID列表
    parent_thought_id: Optional[str]  # 父思想（如果是从另一个思想演化而来）
```

#### 3.1.3 核心算法：即时提炼

```python
def crystallize(self, task_result: TaskResult) -> Optional[ThoughtDiamond]:
    """从任务结果中提炼思想结晶"""
    
    # 1. 提取推理链
    reasoning_steps = self._extract_reasoning_chain(task_result.trace)
    
    # 2. 识别关键决策点
    key_decisions = self._identify_decision_points(task_result.trace)
    
    # 3. 生成结论摘要
    conclusion = self._summarize_conclusion(task_result)
    
    # 4. 提取适用条件
    constraints, assumptions = self._extract_conditions(task_result)
    
    # 5. 计算初始置信度
    confidence = self._calculate_confidence(task_result)
    
    # 6. 组装思想结晶
    thought = ThoughtDiamond(
        thought_id=generate_id(),
        conclusion=conclusion,
        reasoning_chain=reasoning_steps,
        key_decisions=key_decisions,
        constraints=constraints,
        assumptions=assumptions,
        confidence=confidence,
        # ... 其他字段
    )
    
    return thought
```

#### 3.1.4 提炼触发时机

| 触发条件 | 说明 |
|----------|------|
| 任务成功完成 | 提炼成功经验 |
| GoalVerifier 验证通过 | 确认结论可靠 |
| 推理步骤 ≥ 3 步 | 简单问题不提炼（成本>收益） |
| 非重复任务 | 与最近思想相似度 < 0.8 |

#### 3.1.5 与现有模块集成

- **BaseAgent.chat_with_review()**: 在步骤 5（验证通过）后插入结晶调用
- **CheckpointWriter**: 思想结晶纳入检查点快照
- **Dream/Distill**: 结晶作为 Dream 整理的输入单元（不再是原始文本）

### 3.2 Abstraction Hierarchy（抽象层级）

#### 3.2.1 设计目标
- 给每个记忆条目/思想结晶标注抽象层级
- 检索时按任务复杂度匹配对应层级
- 实现"简单问题用浅思想，复杂问题用深思想"

#### 3.2.2 层级定义

```python
class AbstractionLevel(IntEnum):
    """抽象层级定义"""
    L1_FACT = 1        # 事实层：具体数据点、观测值
    L2_INFERENCE = 2   # 推理层：基于事实的直接推论
    L3_STRATEGY = 3    # 策略层：跨场景可复用的方法论
    L4_META_RULE = 4   # 元规则层：关于如何思考的高阶原则
```

#### 3.2.3 层级判定算法

```python
def classify_abstraction(self, content: str, context: Dict) -> AbstractionLevel:
    """判定内容的抽象层级"""
    
    # L4 元规则特征：包含"当...时应该..."、"优先"、"原则"等
    if self._has_meta_pattern(content):
        return AbstractionLevel.L4_META_RULE
    
    # L3 策略特征：可跨场景复用、包含方法论描述
    if self._is_generalizable(content, context):
        return AbstractionLevel.L3_STRATEGY
    
    # L2 推理特征：基于事实的推论、因果关系
    if self._is_inference(content):
        return AbstractionLevel.L2_INFERENCE
    
    # L1 事实：默认层级
    return AbstractionLevel.L1_FACT
```

#### 3.2.4 层级匹配检索

```python
def retrieve_by_complexity(self, query: str, task_complexity: int) -> List[ThoughtDiamond]:
    """按任务复杂度匹配思想层级"""
    
    # 任务复杂度映射到目标层级
    target_level = min(task_complexity, 4)
    
    # 优先检索同层级，降级检索低层级
    results = []
    for level in [target_level, target_level-1, target_level-2, target_level-3]:
        if level < 1:
            break
        level_results = self.memory.retrieve(query, abstraction_level=level)
        results.extend(level_results)
        if len(results) >= self.max_results:
            break
    
    return results[:self.max_results]
```

#### 3.2.5 任务复杂度评估

```python
def assess_task_complexity(self, task: TaskDescription) -> int:
    """评估任务复杂度，返回 1-4"""
    
    score = 1  # 基础分
    
    # 多步骤推理 +1
    if task.requires_multi_step_reasoning:
        score += 1
    
    # 跨领域知识 +1
    if task.cross_domain:
        score += 1
    
    # 需要元认知/策略选择 +1
    if task.requires_meta_reasoning:
        score += 1
    
    return min(score, 4)
```

### 3.3 Confidence Filtering（置信度门控）

#### 3.3.1 设计目标
- 新思想写入记忆前，进行逻辑一致性校验
- 与已有记忆对比，检测矛盾
- 低置信度思想拒绝入库或标记待验证

#### 3.3.2 双重过滤机制

```
新思想 → [置信度过滤] → [冗余过滤] → 入库/拒绝
            │                │
            ├─ 逻辑一致性    ├─ 语义相似度 > 0.9 → 合并
            ├─ 事实正确性    ├─ 相似度 0.7-0.9 → 共存+关联
            └─ 矛盾检测     └─ 相似度 < 0.7 → 独立存储
```

#### 3.3.3 置信度计算

```python
def calculate_confidence(self, thought: ThoughtDiamond) -> float:
    """综合计算思想置信度"""
    
    scores = []
    
    # 1. 逻辑一致性（与已有记忆对比）
    consistency = self._check_logical_consistency(thought)
    scores.append(consistency * 0.35)
    
    # 2. 事实验证（与 Knowledge Library 对比）
    factuality = self._verify_against_knowledge(thought)
    scores.append(factuality * 0.30)
    
    # 3. 来源可信度（GoalVerifier 验证结果）
    source_credibility = self._assess_source(thought)
    scores.append(source_credibility * 0.20)
    
    # 4. 内部一致性（推理链自洽性）
    internal_consistency = self._check_internal_consistency(thought)
    scores.append(internal_consistency * 0.15)
    
    return sum(scores)
```

#### 3.3.4 过滤决策

```python
class FilterDecision(Enum):
    ACCEPT = "accept"           # 置信度 >= 0.7，直接入库
    ACCEPT_WITH_FLAG = "flag"   # 置信度 0.5-0.7，入库但标记"待验证"
    REJECT = "reject"           # 置信度 < 0.5，拒绝入库
    MERGE = "merge"             # 与已有思想高度相似，合并更新
```

#### 3.3.5 与 v4.2 GoalVerifier 集成

```python
# GoalVerifier 验证通过 → 触发置信度过滤
async def verify_and_filter(self, hypothesis: Hypothesis) -> FilterResult:
    """验证后过滤"""
    
    # 1. GoalVerifier 验证（v4.1 已有）
    verification = await self.goal_verifier.verify(hypothesis)
    
    if not verification.passed:
        return FilterResult(FilterDecision.REJECT, "验证未通过")
    
    # 2. 提炼思想（v4.3 Thought Crystallization）
    thought = self.crystallizer.crystallize(verification.result)
    
    # 3. 置信度过滤（v4.3 Confidence Filtering）
    filter_result = self.confidence_filter.filter(thought)
    
    if filter_result.decision == FilterDecision.ACCEPT:
        self.memory.store(thought)
    elif filter_result.decision == FilterDecision.ACCEPT_WITH_FLAG:
        thought.verification_status = "pending"
        self.memory.store(thought)
    # REJECT / MERGE 不入库
    
    return filter_result
```
