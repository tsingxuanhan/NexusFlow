### 1.2 v4.3 缺失的6个维度

| 维度 | 缺失表现 | 核心问题 |
|------|----------|----------|
| **G2 认知架构** | 无认知循环、无工作记忆调度 | Agent被动响应，不能主动规划推理步骤 |
| **G5 因果推理** | 只做相关性分析 | 不能回答"如果X不同会怎样"，不能做干预推理 |
| **G8 元学习** | 碰新领域需重新调研 | 不能"学会怎么学"，不能从少量样本快速适应 |
| **G10 Grounding** | 知识全是符号层面 | 不能将知识接地到具体操作/物理世界 |
| **G11 元认知** | 不知道自己不知道什么 | 不能监控自身认知过程，不能发现理解盲区 |
| **G12 对齐** | 无内化价值判断 | 没有系统性的安全/伦理约束框架 |

### 1.3 v5.0 的核心命题

> **"记忆不是数据的堆砌，而是思想的沉淀；智能不是静态的能力，而是动态的进化。"**

v5.0 不是一个"更大的Agent框架"，而是一个**认知飞轮(Cognitive Flywheel)**——6个缺失维度不是孤立叠加，而是形成一个自我增强的闭环：

```
认知架构(G2) 提供推理框架
    ↓
因果推理(G5) 验证推理正确性
    ↓
元认知(G11) 监控认知过程
    ↓
元学习(G8) 从经验中提取可迁移知识
    ↓
Grounding(G10) 将知识接地到行动
    ↓
对齐(G12) 约束所有行为在安全边界内
    ↓
→ 回到认知架构(G2)，下一轮更精准
```

## 三、各维度详细设计

### 3.1 G2 认知架构：让Agent有"认知循环"

#### 问题诊断
v4.3的Agent是**被动响应式**的——收到任务→执行→返回结果。没有：
- 工作记忆调度（哪些信息在当前推理中活跃）
- 认知循环（感知→推理→行动→学习的闭环）
- 程序性记忆（"怎么做"的技能库）

#### 方案：pyactup + aurora-actr 双层架构

**Layer 1: pyactup (20KB, MIT) — 声明性记忆增强**

pyactup是ACT-R认知架构的轻量Python实现，核心能力：
- **声明性记忆(Declarative Memory)**: 存储事实，带激活值衰减
- **效用方程(Utility Equation)**: 自动学习哪些记忆在什么场景下最有用
- **块化(Chunking)**: 将多个信息单元组合为有意义的块

集成方式：
```python
class CognitiveScheduler:
    """嵌入矿工Agent的认知调度器"""
    
    def __init__(self):
        self.declarative = DeclarativeMemory()  # pyactup声明性记忆
        self.procedural = ProceduralMemory()    # pyactup程序性记忆
        
    def schedule_reasoning(self, task: Task) -> ReasoningPlan:
        """为任务规划认知步骤"""
        # 1. 从声明性记忆中检索相关知识
        activated_chunks = self.declarative.retrieve(
            task.context,
            activation_threshold=0.3  # 激活值阈值
        )
        
        # 2. 匹配程序性记忆中的推理模式
        reasoning_pattern = self.procedural.match_pattern(task.type)
        
        # 3. 生成认知计划
        return ReasoningPlan(
            steps=reasoning_pattern.steps,
            active_memory=activated_chunks,
            estimated_confidence=self._estimate_confidence(activated_chunks)
        )
    
    def update_utility(self, chunk_id: str, success: bool):
        """任务完成后更新记忆效用"""
        self.declarative.update_utility(chunk_id, success)
```

**Layer 2: aurora-actr (MIT, 2026.4发布) — 混合认知引擎**

aurora-actr融合了SOAR+ACT-R，核心能力：
- **块学习(Chunking)**: 从重复推理模式中自动提取新规则
- **MCP桥接**: 天然对接P2P通道
- **生产规则归纳**: 从具体案例归纳通用规则

集成方式：
```python
class AuroraCognitiveEngine:
    """嵌入铸师Agent的高级认知引擎"""
    
    def __init__(self):
        self.engine = AuroraACTR()  # SOAR+ACT-R混合
        self.mcp_bridge = MCPBridge()  # 对接P2P
        
    def chunk_patterns(self, thought_diamonds: List[ThoughtDiamond]):
        """从思想结晶中提取认知模式"""
        for td in thought_diamonds:
            if td.abstraction_level >= 3:  # 只处理高抽象层级的
                self.engine.add_experience(td)
        
        # 触发块学习
        new_rules = self.engine.learn_chunks()
        
        # 新规则写入程序性记忆
        for rule in new_rules:
            self.engine.procedural_memory.add_rule(rule)
```

**RAM预算**: pyactup ~30MB + aurora-actr ~80MB = **~110MB**

### 3.2 G5 因果推理：从"相关"到"因果"

#### 问题诊断
v4.3的试金石Agent只能做**相关性验证**——"A和B同时出现"。不能：
- 判断"A是否导致B"（因果方向）
- 回答"如果A不存在，B还会出现吗"（反事实）
- 从数据中自动发现因果关系（因果发现）

#### 方案：DoWhy + CausalPlan + causal-learn 三层架构

**Layer 1: DoWhy (MIT, PyWhy生态) — 因果验证器**

```python
class CausalVerifier:
    """嵌入试金石Agent的因果验证器"""
    
    def __init__(self):
        self.dowhy = DoWhy()
        
    def verify_causality(self, hypothesis: str, data: pd.DataFrame) -> CausalResult:
        """验证假设的因果性"""
        # 1. 构建因果图
        causal_graph = self.dowhy.causal_graph(
            treatment=hypothesis.cause,
            outcome=hypothesis.effect,
            common_causes=self._identify_confounders(data)
        )
        
        # 2. 识别因果效应
        identified = self.dowhy.identify_effect(causal_graph)
        
        # 3. 估计因果效应
        estimate = self.dowhy.estimate_effect(
            identified,
            method_name="backdoor.propensity_score_matching"
        )
        
        # 4. 稳健性检验
        robust = self.dowhy.refute_estimate(
            identified, estimate,
            method_name="random_common_cause"
        )
        
        return CausalResult(
            causal_strength=estimate.value,
            is_causal=abs(estimate.value) > 0.3 and robust.is_robust,
            confounders=self._identify_confounders(data),
            counterfactual=self._compute_counterfactual(hypothesis, data)
        )
```

### 3.3 G8 元学习：让系统"学会怎么学"

#### 问题诊断
v4.3碰到全新领域时：
- 需要从0开始调研（无法复用之前的学习经验）
- 新知识可能覆盖旧知识（灾难性遗忘）
- 不能从少量样本快速适应

#### 方案：EWC + MetaAgent + GEPA自进化管道

**Layer 1: EWC (Elastic Weight Consolidation) — 权重保护**

```python
class EWCProtectedMemory:
    """在NGramTFIDFProvider上叠加EWC保护"""
    
    def __init__(self, base_provider: NGramTFIDFProvider):
        self.base = base_provider
        self.fisher_info = {}  # Fisher信息矩阵
        self.old_weights = {}  # 旧权重快照
        
    def compute_fisher(self, task_results: List[TaskResult]):
        """从任务结果中计算Fisher信息"""
        for result in task_results:
            for chunk_id, weight in result.affected_weights.items():
                if chunk_id not in self.fisher_info:
                    self.fisher_info[chunk_id] = 0.0
                # Fisher信息 = 梯度平方的期望
                self.fisher_info[chunk_id] += weight ** 2
        
    def regularized_update(self, chunk_id: str, new_weight: float) -> float:
        """带EWC正则化的权重更新"""
        if chunk_id in self.fisher_info:
            old_w = self.old_weights.get(chunk_id, new_weight)
            fisher = self.fisher_info[chunk_id]
            # EWC正则化：重要权重变化越小越好
            penalty = fisher * (new_weight - old_w) ** 2
            return new_weight - 0.01 * penalty  # 正则化系数
        return new_weight
    
    def snapshot_weights(self):
        """任务完成后快照当前权重"""
        self.old_weights = self.base.get_all_weights().copy()
```

**Layer 2: MetaAgent — 自演化机制**

```python
class MetaLearningAgent:
    """从经验中自动提取可迁移知识"""
    
    def __init__(self):
        self.tool_library = {}  # 已验证的工具/方法
        self.transfer_rules = []  # 可迁移规则
        
    def distill_experience(self, task_results: List[TaskResult]):
        """从任务结果中蒸馏经验"""
        for result in task_results:
            if result.success:
                # 提取成功模式
                pattern = self._extract_success_pattern(result)
                self._add_to_tool_library(pattern)
            else:
                # 提取失败教训
                lesson = self._extract_failure_lesson(result)
                self.transfer_rules.append(lesson)
    
    def generate_new_tool(self, task_type: str) -> Optional[Tool]:
        """为新任务类型自动生成工具"""
        # 检索相似任务的工具
        similar_tools = self._retrieve_similar(task_type)
        
        if similar_tools:
            # 组合现有工具
            return self._compose_tools(similar_tools, task_type)
        else:
            # 从迁移规则中推导
            applicable_rules = [r for r in self.transfer_rules if r.applies_to(task_type)]
            if applicable_rules:
                return self._derive_tool(applicable_rules, task_type)
        return None
```

**Layer 3: GEPA自进化管道 — 基于DSPy**

```python
class GEPASelfEvolution:
    """遗传-Pareto反思优化：自动进化Agent提示和技能"""
    
    def __init__(self):
        self.dspy = DSPy()
        self.optimizer = GEPAOptimizer()
        
    def evolve_agent_prompts(self, agent_role: str, task_dataset: List[Task]):
        """进化指定角色的系统提示"""
        # 1. 收集执行轨迹
        traces = self._collect_traces(agent_role, task_dataset)
        
        # 2. 诊断失败原因
        failures = [t for t in traces if not t.success]
        diagnosis = self._diagnose_failures(failures)
        
        # 3. 生成改进候选（遗传算法）
        candidates = self.optimizer.generate_candidates(
            current_prompt=self._get_prompt(agent_role),
            diagnosis=diagnosis,
            population_size=10
        )
        
        # 4. Pareto评估（准确率 + token效率）
        best = self.optimizer.pareto_select(candidates, task_dataset)
        
        # 5. 反思改进
        refined = self.optimizer.reflect(best, failures)
        
        return refined
    
    def evolve_skill(self, skill_path: str) -> Optional[str]:
        """进化SKILL.md技能文件"""
        # 读取执行轨迹
        traces = self._load_skill_traces(skill_path)
        
        # 调用Hermes Self-Evolution模式
        improved = self.dspy.optimize(
            program=skill_path,
            metric=self._skill_metric,
            optimizer="GEPA"
        )
        
        return improved
```

### 3.4 G11 元认知：知道自己"不知道什么"

#### 问题诊断
v4.3的Agent：
- 不能监控自己的置信度是否合理
- 不能发现自己的理解盲区
- 不能识别认知偏差（确认偏差、锚定效应等）

#### 方案：Reflexion + Vibe Check + PSR 三层元认知

**Layer 1: Reflexion — 反思循环**

```python
class ReflexionEngine:
    """语言反思：失败→语言反馈→自我修正"""
    
    def __init__(self):
        self.episodic_memory = []  # 反思经验存储
        
    def reflect(self, task_result: TaskResult) -> ReflexionResult:
        """对任务结果进行反思"""
        if task_result.success:
            return ReflexionResult(should_retry=False)
        
        # 1. 生成语言反馈
        feedback = self._generate_feedback(task_result)
        
        # 2. 存储反思经验
        self.episodic_memory.append(ReflexionEpisode(
            task=task_result.task,
            failure_reason=feedback.failure_analysis,
            corrective_action=feedback.suggested_fix,
            timestamp=time.time()
        ))
        
        # 3. 判断是否值得重试
        if len(self.episodic_memory) < 3:  # 最多反思3次
            return ReflexionResult(
                should_retry=True,
                feedback=feedback,
                similar_past_episodes=self._retrieve_similar(task_result.task)
            )
        
        return ReflexionResult(should_retry=False, give_up_reason="max_reflections")
    
    def _generate_feedback(self, result: TaskResult) -> Feedback:
        """生成语言形式的反思反馈"""
        prompt = f"""
你是一个科学研究的元认知监控器。以下任务失败了：
- 任务: {result.task.description}
- 尝试: {result.attempt_summary}
- 错误: {result.error_summary}

请分析：
1. 失败的根本原因是什么？（不是表面原因）
2. 哪种策略可能解决这个问题？
3. 过去是否有类似的成功案例可以参考？
"""
        return self._call_llm(prompt)
```

**Layer 2: Vibe Check — 置信度监控**

```python
class VibeCheckMonitor:
    """元认知中断器：实时监控Agent置信度"""
    
    def __init__(self):
        self.confidence_history = {}  # 每个Agent的置信度历史
        self.bias_detectors = self._init_bias_detectors()
        
    def monitor(self, agent_output: AgentOutput, agent_role: str) -> MonitorResult:
        """监控Agent输出的置信度"""
        # 1. 评估置信度
        confidence = self._estimate_confidence(agent_output)
        
        # 2. 与历史对比
        history = self.confidence_history.get(agent_role, [])
        if history and abs(confidence - np.mean(history)) > 0.3:
            # 置信度异常波动
            return MonitorResult(
                flag="CONFIDENCE_ANOMALY",
                action="PAUSE_AND_REFLECT"
            )
        
        # 3. 检测认知偏差
        biases = self._detect_biases(agent_output)
        if biases:
            return MonitorResult(
                flag="BIAS_DETECTED",
                biases=biases,
                action="REQUEST_SECOND_OPINION"
            )
        
        # 4. 记录历史
        history.append(confidence)
        self.confidence_history[agent_role] = history[-50:]  # 保留最近50条
        
        return MonitorResult(flag="NORMAL")
    
    def _detect_biases(self, output: AgentOutput) -> List[Bias]:
        """检测认知偏差"""
        biases = []
        
        # 确认偏差：只关注支持假设的证据
        if self.bias_detectors["confirmation"].check(output):
            biases.append(Bias.CONFIRMATION)
        
        # 锚定效应：过度依赖第一个信息
        if self.bias_detectors["anchoring"].check(output):
            biases.append(Bias.ANCHORING)
        
        # 可得性偏差：过度依赖容易想到的例子
        if self.bias_detectors["availability"].check(output):
            biases.append(Bias.AVAILABILITY)
        
        return biases
```

**Layer 3: PSR — 渐进式自反思安全层**

```python
class ProgressiveSelfReflection:
    """推理时的自监控和纠正"""
    
    def __init__(self, base_llm):
        self.base_llm = base_llm
        self.reflection_predictor = ReflectionRoundPredictor()
        
    def generate_with_reflection(self, prompt: str, max_rounds: int = 3) -> str:
        """带渐进式自反思的生成"""
        output = self.base_llm.generate(prompt)
        
        for round_idx in range(max_rounds):
            # 1. 自反思：这个输出有什么问题？
            self_critique = self.base_llm.generate(
                f"Critique the following output for errors, biases, or gaps:\n{output}"
            )
            
            # 2. 判断是否需要修正
            if self._is_satisfactory(self_critique):
                break
            
            # 3. 修正
            output = self.base_llm.generate(
                f"Original: {output}\nCritique: {self_critique}\nRevised output:"
            )
            
            # 4. 预测是否还需要更多轮次
            predicted_rounds = self.reflection_predictor.predict(output, round_idx)
            if predicted_rounds <= round_idx + 1:
                break
        
        return output
```

### 3.5 G10 Grounding：将知识接地到行动

#### 问题诊断
v4.3的知识全是**符号层面**的——知道"SiO₂掺量与强度正相关"，但不知道：
- 如何在实验中实际操作
- 如何在GUI界面上执行任务

#### 方案：Voyager技能库模式 + Agent S3 GUI接地

**Layer 1: Voyager技能库 — 代码即技能**

```python
class SkillLibrary:
    """Voyager式技能库：技能=可执行代码+向量索引"""
    
    def __init__(self):
        self.skills = {}  # skill_id → Skill
        self.skill_index = VectorIndex()  # 技能向量索引
        
    def register_skill(self, task: Task, solution_code: str, success: bool):
        """注册新技能"""
        if not success:
            return
        
        skill = Skill(
            skill_id=generate_id(),
            description=task.description,
            code=solution_code,
            preconditions=self._extract_preconditions(task),
            postconditions=self._extract_postconditions(task),
            embedding=embed(task.description)
        )
        
        self.skills[skill.skill_id] = skill
        self.skill_index.add(skill)
    
    def retrieve_skills(self, task_description: str, top_k: int = 5) -> List[Skill]:
        """检索相关技能"""
        similar = self.skill_index.search(embed(task_description), top_k)
        return [self.skills[s.id] for s in similar]
    
    def compose_skills(self, task: Task) -> Optional[str]:
        """组合多个技能完成复杂任务"""
        # 1. 分解任务为子任务
        subtasks = self._decompose_task(task)
        
        # 2. 为每个子任务检索技能
        skill_sequences = []
        for subtask in subtasks:
            skills = self.retrieve_skills(subtask.description)
            if skills:
                skill_sequences.append(skills[0].code)
            else:
                return None  # 某个子任务没有匹配技能
        
        # 3. 组合代码
        return self._compose_code(skill_sequences, subtasks)
```

```python
class GUIAgent:
    """GUI操作Agent：将知识转化为界面操作"""
    
    def __init__(self):
        self.screen_encoder = ScreenEncoder()
        self.action_planner = ActionPlanner()
        
    def execute_task(self, task_description: str, app_context: str) -> ActionResult:
        """通过GUI操作执行任务"""
        # 1. 截图当前屏幕
        screenshot = self._capture_screen()
        
        # 2. 编码屏幕内容
        screen_repr = self.screen_encoder.encode(screenshot)
        
        # 3. 规划操作序列
        actions = self.action_planner.plan(
            task=task_description,
            screen_state=screen_repr,
            app_context=app_context
        )
        
        # 4. 执行操作
        for action in actions:
            self._execute_action(action)
            # 验证每步结果
            if not self._verify_action_result(action):
                return ActionResult(success=False, failed_at=action)
        
        return ActionResult(success=True)
```

### 3.6 G12 对齐：内化价值判断

#### 问题诊断
v4.3没有系统性的安全/伦理约束：
- Agent可能产出有害建议
- 没有内化的价值判断框架
- 无法自我批评和纠正偏差

#### 方案：Constitutional AI + ArGen Policy-as-Code

**Layer 1: Constitutional AI — 宪法原则**

```python
class ConstitutionalAIChecker:
    """AI自我批评+修正的对齐框架"""
    
    def __init__(self):
        self.principles = [
            "输出不应包含对任何群体的歧视或偏见",
            "科学结论应基于证据，不应过度推断",
            "涉及安全的建议应保守，宁可多警告",
            "不应帮助创建危险物质或武器",
            "应承认不确定性，不应伪装确定性",
            "应尊重知识产权，标注引用来源",
            "涉及人类受试者的研究应遵循伦理规范",
            "环境相关建议应考虑可持续性",
            "应平衡创新与风险",
            "对超出能力范围的问题应诚实说明"
        ]
        
    def check_and_revise(self, agent_output: AgentOutput) -> RevisedOutput:
        """检查并修正输出"""
        # 1. 自我批评
        critique = self._self_critique(agent_output.content)
        
        if critique.violations:
            # 2. 修正
            revised = self._revise(
                original=agent_output.content,
                violations=critique.violations,
                principles=self.principles
            )
            return RevisedOutput(
                content=revised,
                was_modified=True,
                modifications=critique.violations
            )
        
        return RevisedOutput(content=agent_output.content, was_modified=False)
    
    def _self_critique(self, content: str) -> Critique:
        """AI自我批评"""
        prompt = f"""
请检查以下内容是否违反以下原则：
{self._format_principles()}

内容：{content}

对每条原则，判断是否违反，如果违反请指出具体位置和原因。
"""
        return self._call_llm(prompt)
```

**Layer 2: ArGen Policy-as-Code — 可编程约束**

```python
class PolicyEngine:
    """策略引擎：可编程的Agent行为约束"""
    
    def __init__(self):
        self.policies = self._load_policies()
        
    def check_policy(self, action: AgentAction, agent_role: str) -> PolicyResult:
        """检查Action是否违反策略"""
        for policy in self.policies:
            if policy.applies_to(agent_role, action):
                result = policy.evaluate(action)
                if not result.allowed:
                    return PolicyResult(
                        blocked=True,
                        reason=result.reason,
                        suggested_alternative=result.alternative
                    )
        
        return PolicyResult(blocked=False)
    
    def add_policy(self, policy_yaml: str):
        """动态添加策略（无需重新部署）"""
        policy = Policy.from_yaml(policy_yaml)
        self.policies.append(policy)
```
